Guia Completo: Podman & podman-compose no Linux

Do básico ao avançado — containers rootless, pods, redes, volumes e orquestração.

Seções

O que é o Podman?

Podman é um motor de containers daemonless e rootless compatível com OCI. Substitui o Docker sem precisar de um daemon rodando em background — cada container é um processo filho direto do shell.

CaracterísticaDockerPodman
Daemon necessário✅ dockerd❌ Nenhum
Rootless nativoParcial✅ Total
Suporte a Pods❌✅ (como K8s)
CLI compatível c/ Docker—✅ Drop-in
Geração de unit systemd❌✅
Composedocker-composepodman-compose
Dica: Você pode criar um alias alias docker=podman no seu ~/.bashrc — quase todos os comandos são idênticos.

Instalação

Ubuntu / Debian
bash
# Ubuntu 22.04+
sudo apt update
sudo apt install -y podman podman-compose

# Verificar versão
podman --version
podman-compose --version
Fedora / RHEL / CentOS
bash
sudo dnf install -y podman podman-compose
Arch Linux
bash
sudo pacman -S podman podman-compose
Configuração Rootless (após instalação)
bash
# Verificar subuid/subgid (necessário para rootless)
cat /etc/subuid
cat /etc/subgid

# Se o usuário não aparecer, adicionar:
sudo usermod --add-subuids 100000-165535 $USER
sudo usermod --add-subgids 100000-165535 $USER

# Habilitar linger (container sobrevive ao logout)
loginctl enable-linger $USER

Comandos Básicos

Executar containers
bash
# Hello world
podman run hello-world

# Interativo com terminal
podman run -it ubuntu bash

# Em background (detached) com nome
podman run -d --name meu-nginx -p 8080:80 nginx

# Com variáveis de ambiente
podman run -d --name meu-postgres \
  -e POSTGRES_PASSWORD=senha123 \
  -e POSTGRES_DB=meubanco \
  -p 5432:5432 postgres:16

# Remover container ao parar
podman run --rm alpine echo "temporário"
Gerenciar containers
bash
podman ps               # Listar rodando
podman ps -a             # Listar todos
podman stop meu-nginx   # Parar
podman start meu-nginx  # Iniciar
podman restart meu-nginx
podman rm meu-nginx     # Remover (parado)
podman rm -f meu-nginx  # Forçar remoção

# Logs
podman logs meu-nginx
podman logs -f meu-nginx  # Follow

# Executar comando em container rodando
podman exec -it meu-nginx bash

# Inspecionar
podman inspect meu-nginx
podman stats             # Uso de recursos (live)

Imagens & Containerfile

Gerenciar imagens
bash
podman images                      # Listar imagens
podman pull nginx:alpine           # Baixar imagem
podman rmi nginx:alpine            # Remover imagem
podman image prune                 # Limpar imagens órfãs
podman search nginx               # Buscar no registry

# Registries alternativos
podman pull quay.io/fedora/fedora
podman pull ghcr.io/user/imagem:tag
Exemplo: Containerfile (Node.js app)
Containerfile
# Containerfile é o nome padrão no Podman
FROM node:20-alpine

WORKDIR /app

# Copiar deps primeiro (cache eficiente)
COPY package*.json ./
RUN npm ci --omit=dev

COPY . .

# Usuário não-root (boa prática)
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

EXPOSE 3000
CMD ["node", "server.js"]
Build multi-stage (produção)
Containerfile
# Estágio 1: build
FROM golang:1.22 AS builder
WORKDIR /src
COPY . .
RUN CGO_ENABLED=0 go build -o app .

# Estágio 2: imagem final mínima
FROM scratch
COPY --from=builder /src/app /app
ENTRYPOINT ["/app"]
Build e tag
bash
# Construir imagem
podman build -t minha-app:1.0 .

# Usar Containerfile com nome diferente
podman build -f Containerfile.prod -t minha-app:prod .

# Tag adicional e push
podman tag minha-app:1.0 quay.io/user/minha-app:1.0
podman push quay.io/user/minha-app:1.0

# Exportar/importar imagem como arquivo
podman save -o app.tar minha-app:1.0
podman load -i app.tar

Volumes & Redes

Volumes
bash
# Criar e usar volume nomeado
podman volume create pgdata
podman run -d --name postgres \
  -v pgdata:/var/lib/postgresql/data \
  -e POSTGRES_PASSWORD=senha \
  postgres:16

# Bind mount (pasta do host)
podman run -v /home/user/dados:/data:Z alpine ls /data
# :Z aplica label SELinux (importante no RHEL/Fedora!)

# Volume somente leitura
podman run -v /etc/config:/config:ro nginx

# Gerenciar volumes
podman volume ls
podman volume inspect pgdata
podman volume rm pgdata
podman volume prune       # Limpar não utilizados
No SELinux (Fedora/RHEL), sempre use :Z em bind mounts de diretórios do host para evitar erros de permissão.
Redes
bash
# Criar rede bridge
podman network create minha-rede

# Containers na mesma rede se comunicam pelo nome
podman run -d --name db --network minha-rede \
  -e POSTGRES_PASSWORD=senha postgres:16

podman run -d --name api --network minha-rede \
  -e DATABASE_URL=postgresql://postgres:senha@db:5432/app \
  minha-api:latest

# Conectar container existente a uma rede
podman network connect minha-rede meu-container

# Inspecionar rede
podman network ls
podman network inspect minha-rede

# Rede host (sem isolamento, máx performance)
podman run --network host nginx

Pods (como Kubernetes)

Pods agrupam containers que compartilham namespace de rede e podem compartilhar volumes — exatamente como Pods do Kubernetes.

Criar e usar pods
bash
# Criar pod com porta exposta
podman pod create --name webapp -p 8080:80

# Adicionar containers ao pod
podman run -d --pod webapp --name nginx nginx:alpine
podman run -d --pod webapp --name app minha-api

# Dentro do pod, containers se comunicam via localhost!
# O nginx pode fazer proxy para localhost:3000 (a api)

# Gerenciar pods
podman pod ls
podman pod stop webapp
podman pod start webapp
podman pod rm -f webapp    # Remove pod e containers

# Estatísticas do pod completo
podman pod stats webapp
Gerar YAML Kubernetes a partir de um pod
bash
# Exportar pod como manifesto K8s
podman generate kube webapp -f webapp.yaml

# Aplicar manifesto K8s com Podman (sem K8s!)
podman play kube webapp.yaml

# Remover o que foi criado pelo manifesto
podman play kube --down webapp.yaml

podman-compose

O podman-compose é compatível com docker-compose.yml. Basta apontar para o mesmo arquivo.

Exemplo: Stack Web completa (compose.yml)
yaml
version: "3.9"

services:
  postgres:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: appdb
      POSTGRES_USER: appuser
      POSTGRES_PASSWORD: senhasecreta
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U appuser"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    command: redis-server --maxmemory 256mb

  api:
    build:
      context: ./api
      dockerfile: Containerfile
    restart: unless-stopped
    environment:
      DATABASE_URL: postgresql://appuser:senhasecreta@postgres:5432/appdb
      REDIS_URL: redis://redis:6379
      NODE_ENV: production
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started
    expose:
      - "3000"

  nginx:
    image: nginx:alpine
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - certdata:/etc/letsencrypt
    depends_on:
      - api

volumes:
  pgdata:
  certdata:

networks:
  default:
    name: webapp-net
Comandos podman-compose
bash
# Subir toda a stack
podman-compose up -d

# Subir apenas um serviço
podman-compose up -d postgres

# Rebuild e subir
podman-compose up -d --build

# Logs de todos os serviços
podman-compose logs -f

# Executar comando em serviço
podman-compose exec api sh
podman-compose exec postgres psql -U appuser appdb

# Parar sem remover
podman-compose stop

# Parar e remover containers
podman-compose down

# Parar, remover containers E volumes
podman-compose down -v

# Escalar serviço (réplicas)
podman-compose up --scale api=3 -d

Recursos Avançados

Limites de recursos
bash
# Limitar CPU e memória
podman run -d \
  --cpus 1.5 \
  --memory 512m \
  --memory-swap 1g \
  --name app-limitado \
  minha-app

# Prioridade de CPU
podman run --cpu-shares 512 minha-app  # padrão=1024
Secrets (senhas seguras)
bash
# Criar secret a partir de stdin
echo "minha-senha-super-segura" | podman secret create db_password -

# Criar secret a partir de arquivo
podman secret create ssl_cert /path/to/cert.pem

# Usar secret no container (montado em /run/secrets/)
podman run -d --secret db_password postgres:16

# Listar e remover secrets
podman secret ls
podman secret rm db_password
Copiar arquivos
bash
# Host → Container
podman cp ./config.json meu-app:/app/config.json

# Container → Host
podman cp meu-app:/app/logs/app.log ./app.log
Inspecionar e debugar
bash
# Ver IP e redes do container
podman inspect --format '{{.NetworkSettings.IPAddress}}' meu-app

# Diff do sistema de arquivos vs imagem base
podman diff meu-app

# Commitar estado atual como nova imagem
podman commit meu-app minha-app-modificada:v2

# Limpar tudo (containers, imagens, volumes parados)
podman system prune -a --volumes

# Informações do sistema
podman system info
podman system df     # Uso de disco

Integração com Systemd

Podman pode gerar units do systemd para que containers iniciem automaticamente com o sistema — sem daemon, nativo no Linux.

Gerar unit systemd (container)
bash
# Criar container que vai virar serviço
podman run -d --name meu-nginx -p 8080:80 nginx

# Gerar arquivo de unit systemd
podman generate systemd --name meu-nginx \
  --files --new

# Mover para diretório do usuário (rootless)
mkdir -p ~/.config/systemd/user/
mv container-meu-nginx.service ~/.config/systemd/user/

# Habilitar e iniciar
systemctl --user daemon-reload
systemctl --user enable --now container-meu-nginx.service

# Verificar status
systemctl --user status container-meu-nginx.service
Quadlet (modo moderno — Podman 4.4+)
Quadlets são arquivos declarativos que o systemd usa para gerar units automaticamente. É a forma recomendada atualmente.
~/.config/containers/systemd/nginx.container
[Unit]
Description=Nginx via Podman Quadlet
After=network-online.target

[Container]
Image=nginx:alpine
PublishPort=8080:80
Volume=/home/user/html:/usr/share/nginx/html:Z
Environment=NGINX_HOST=meusite.com
AutoUpdate=registry

[Service]
Restart=always
TimeoutStartSec=30

[Install]
WantedBy=default.target
Ativar Quadlet
bash
# Recarregar systemd (detecta os .container automaticamente)
systemctl --user daemon-reload

# Habilitar e iniciar
systemctl --user enable --now nginx.service

# Auto-update de imagens (via podman-auto-update)
systemctl --user enable podman-auto-update.timer
Recomendação: Para novas instalações use Quadlets (.container files) em vez do podman generate systemd — são mais simples, declarativos e suportados ativamente.
